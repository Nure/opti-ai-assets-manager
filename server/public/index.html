2. Download assets

Download source assets to the local machine where the Docker app is running. Files are written directly into ./Downloaded on date and time in the app directory through the Docker volume. Nested CMP/XLSX folders are preserved directly inside that timestamped root folder.

Option A — Download from XLSX public link

Use a public XLSX manifest containing asset URLs. The app downloads files in parallel, preserves folder paths, calculates checksum, and creates a separate download report.

Download sample XLSX template

Supported columns: source_url, asset_url, download_url, cmp_url, url, file_name, folder_path, title, asset_id, asset_guid.

Option B — Download from CMP DAM

Generate an OAuth bearer token from Client ID/Secret, then Assets by Folder ID or All Assets. CMP folder listing uses offset/page_size pagination and downloads 10 assets at a time.

Analyze modes generate XLSX reports without downloading binaries. Metadata mode also calls GET /assets/{asset_id}/fields for labels/field values.

Retry policy: each file download retries up to 3 times. CMP API list/get/fields calls refresh the bearer token and retry once on 401/403.

4. Tag Assets Metadata

Validate an XLSX metadata sheet against CMP fields and labels, perform a mandatory dry run, then execute safe per-field updates for each asset.

CMP authentication

Uses the same CMP OAuth client credentials flow as Download Assets and Import & Migrate.

Spreadsheet

Format: AssetID, then one column per CMP field or label. Multi-value cells can use semicolon separators.

No XLSX selected

Dry Run uses the currently selected XLSX only. If a dry run fails, remove the file, fix the spreadsheet, and upload the corrected XLSX.

Execution is disabled until a dry run has produced validated asset-field mappings.

Dry Run and execution safety

Dry Run fetches CMP fields and label groups, maps XLSX headers to CMP field/label IDs, resolves dropdown/label option text to IDs, and reports missing fields or invalid values before any update is sent.
